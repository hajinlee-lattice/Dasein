Authoring Talking Points

Talking points provide messages that enable sales reps to successfully engage with customers. Adding dynamic, customer-specific talking points gives your reps additional context.

Choosing a Language

  • Click the dropdown menu for Viewing Language and select from the list.

    Note:

    playMAKER groups talking points by language. You can add talking points for as many languages as needed. The number in parentheses next to the language name indicates the number of talking points that exist for this language.

    If you add talking points in different languages, at runtime the system matches language-specific talking points with users' locales.

    You cannot change the language if any talking point on the page has a validation error.

Adding a Talking Point

  • Click Add Talking Point to add a new talking point.

Saving Talking Points

  • Save talking points by doing any of the following:
    • Click I'm Finished.
    • Click outside of the Content tab area (for example, select All Plays).
    • Click any of the other play tabs.

Pasting Text

Plain text mode is recommended for pasting talking point syntax examples. This mode converts unsupported newline symbols to <p> tags so that the text passes validation. To add text from Microsoft Word and preserve formatting (such as colors), paste directly into the editor (using CTRL+V). If your text contains both dynamic syntax and styling, you will need to paste as plain text to preserve the dynamic syntax and then re-add styling using the provided tools.

  • Click the icon to enter plain text mode.

Checking Spelling

  • Click the icon to check for spelling errors.

    Note:

    • The spell checker does not validate attribute names. The editor shows a red outline if any syntax is incorrect.
    • English is the only language that supports spell-checking.
    • Microsoft Internet Explorer does not support this spell checker.

Styling Text

  • Click the Font Size menu to change the size of text.
  • Click the underlined 'A' to change the color of text.

Editing HTML Source

  • Click the icon to edit any HTML source.
    Talking points support the following tags:
    • <a[href]> target
    • <b>
    • <br>
    • <font[color|face|size]>
    • <img[src|id|width|height|align|hspace|vspace]>
    • <i>
    • <li>
    • <p[align|class]>
    • <h1> through <h6>
    • <span[class]>
    • <textformat[blockindent|indent|leading|leftmargin|rightmargin|tabstops]>
    • <u>

    The Title field does not support HTML markup. If this field contains HTML or tags that the editor interprets as such, the output will be encoded as in the following example:

    • “<p>Competitive Differentiation</p>”  -> "&lt;p&gt;Competitive Differentiation&lt;/p&gt;"

Adding Links

  • You can add links to talking points, such as additional resources to help reps sell to accounts.
    1. Type / paste and then highlight the text that you would like to make a link. This text may say anything, such as "Click here".
    2. Click the icon.
    3. In the Insert/Edit Link dialog:
      1. Paste or type the web address into the Link URL field.
      2. Click the Target dropdown and choose whether clicking this link should Open Link in the Same Window or Open Link in a New Window.
      3. For Title, enter any text that should display as a tooltip when a user places the cursor over the link.
      4. Click Insert to add the link to your talking point, or Cancel.
    4. To remove a link, highlight the linked text and click the icon.

Adding Attributes

  • Drag talking point attributes from the Attributes menu at the right onto the body text field.

Collapsing a Talking Point

  • Collapse and expand talking points as needed while you edit your play's messaging. You can collapse and expand talking points even if they are invalid.
    1. Click the button next to an expanded talking point to collapse it.
    2. Click the button next to a collapsed talking point to expand it.

Reordering Talking Points

  • Click and drag talking points to reorder them.

    You can drag talking points whether they are expanded or collapsed.

    You can reorder talking points even if they are invalid.

    All talking points on the page collapse and expand again when you drop the talking point.

Validating a Talking Point

  • Be sure to resolve any invalid talking poiAnts.

    playMAKER validates each talking point when you click outside of that talking point's editor.

    If you have invalid talking points, you cannot save the play or make changes to any aspect of the play outside of the Content tab until you resolve the issues. You may continue to add and edit new talking points.

Deleting a Talking Point

  • Click the button beside a talking point to delete it.

    You can delete a talking point whether it is valid or invalid.

Displaying Dynamic Talking Points

  • Dynamic talking points make use of variables and conditional expressions. These variables and expressions rely on special syntax to display the actual values correctly.

    1. Braces must enclose each attribute name. Attributes that you drag into the talking point editor automatically display this syntax. Within the braces, an exclamation point precedes the full attribute name. If it is an account attribute, a period separates the object name from the attribute (column) name.

      Reference extensions using the same syntax as described above. For example, you would reference an account's revenue growth using {!Account.RevenueGrowth}.

    2. Compare values using the following operators:
      • = Equal to
      • > Greater than
      • < Less than
      • > = Greater than or equal to
      • <= Less than or equal to
      • < > Not equal to

      Compare any string values by enclosing these in quotes: [{!Objecttype.Column} Operator 'Value']

    3. Compare boolean attribute values using yes and no or true and false.

    4. Square brackets must enclose each conditional expression.

      Any text you wish to display when a conditional expression is true must appear on the line below the conditional expression. You cannot mix a conditional expression and display text on the same line. You can, however, include variables (embedded replacement tokens like {!Today}) on the same line as display text.

      The phrase [end] must appear after each conditional expression block.

      Examples:

      [{!Account.StorageBudget} > 5000]

      This account's storage budget is ${!Account.StorageBudget}. This customer may consider purchasing a SAN this quarter.

      [end]

      [{!Account.IsRecentCIOTransition} = yes]

      This account has a new CIO who is likely assessing the company's technology needs and may be looking for ways to improve the company's infrastructure. Consider starting a conversation about upgrades with contacts at this account. If they have been purchasing storage solutions from another vendor until now, they may be eligible for a special discount if they choose to switch to one of our solutions.

      [end]

    5. Use 'AND' and 'OR' between statements to conditionally display talking points that meet multiple criteria.

      Examples:

      [{!Account.JobPostings} > 5 OR {!Account.Funding} > 0]

      {!Space}

      Current data reveals that {!Account.DisplayName} is experiencing rapid growth.

      [end]

      [{!Account.JobPostings} > 5]

      This customer currently lists {!Account.JobPostings} employment opportunities.

      [end]

      [{!Account.Funding} > 0]

      This customer has recently received ${!Account.Funding} in private funding.

      [end]

      Companies at this stage of growth may need additional servers, or might want to consider virtualization.

      [end]

    6. Use pairs of parentheses to express the order of operations.

      Example:

      [({!Account.Vertical} = 'Health' OR {!Account.Vertical} = 'Financial') AND ({!Account.StorageBudget} > 5000)]

      Current data shows that {!Account.DisplayName} has high likelihood of purchasing one of our storage products this quarter. Healthcare and financial services companies tend to require more robust backup and recovery solutions.

      [end]
